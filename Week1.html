Backend (.NET Core) 1. How do you handle Global Exception Handling in ASP.NET Core?
I use Custom Middleware or the UseExceptionHandler extension to catch exceptions globally, log them, and return a standardized JSON error format.

Backend (C#) 2. What is the difference between IQueryable and IEnumerable?
IQueryable executes filters on the Database side (SQL Server), making it better for large datasets. IEnumerable loads data into memory first and then filters.
Real-time Example: I consistently use IQueryable with LINQ in my projects to ensure only the necessary records are fetched, optimizing memory usage[cite: 1].
Security 3. Why use JWT over Session-based authentication?
JWT is stateless, meaning the server doesn't need to store session data, which allows the application to scale horizontally across multiple servers/Azure App Services easily.
Real-time Example: For my E-commerce project, I used JWT to ensure that as traffic scaled, the authentication remained fast and didn't burden the Azure SQL database[cite: 1].
Azure 4. How do you manage application secrets in Azure?
I use Azure Key Vault to store connection strings and API keys. The application retrieves these at runtime using Managed Identities, so no secrets are stored in code.
Real-time Example: As part of my AZ-204 practice, I integrated Key Vault into our CI/CD pipeline to ensure production secrets are never visible to developers in GitHub/Azure DevOps[cite: 1].
Database 5. How did you achieve a 30% performance improvement in SQL?
I focused on Stored Procedure optimization: identifying missing indexes, avoiding cursors, and reducing unnecessary joins or "Select *" queries.
Real-time Example: At Cognine, I refactored legacy SQL functions that were slowing down report generation, resulting in a 30% faster response time for the end-users[cite: 1].
Frontend (Angular) 6. Explain the concept of Lazy Loading in Angular.
Lazy loading loads feature modules only when the user navigates to their routes, which reduces the initial bundle size and speeds up the first page load.
Real-time Example: I implemented lazy loading for the "Admin Panel" of the E-commerce app, so regular customers didn't have to download heavy admin-related code[cite: 1].
Tools & SDKs 7. How do you handle large-scale Word document generation?
I use the Open XML SDK to interact with the XML structure of .docx files directly, which is faster and more reliable than using Interop services on a server.
Real-time Example: In the ELOGR project, this method allowed me to automate report generation, reducing the time taken by 70%[cite: 1].
Frontend (jQuery/Ajax) 8. When would you choose jQuery over a framework like Angular?
jQuery is excellent for lightweight DOM manipulation or enhancing existing MVC views without the overhead of a full SPA framework.
Real-time Example: For the RAB application's existing modules, I used jQuery and Ajax to add asynchronous features without needing to rewrite the entire legacy frontend in Angular[cite: 1].
Architecture 9. What is the benefit of using an API Gateway?
It acts as a single entry point for microservices, handling cross-cutting concerns like authentication, rate limiting, and request routing.
Real-time Example: My research into backend architecture includes service-to-service communication patterns and bounded contexts to keep microservices decoupled[cite: 1].
DevOps 10. What is a CI/CD pipeline in your experience?
It is an automated process where CI (Continuous Integration) builds and tests the code, and CD (Continuous Deployment) pushes it to environments like Azure App Service.
Real-time Example: I used Azure DevOps to automate our deployment cycle, ensuring that every bug fix was automatically tested and deployed to the staging environment[cite: 1].
Framework Internals 11. What is the difference between .NET Framework and .NET Core?
.NET Framework is Windows-only, while .NET Core is cross-platform, modular, and optimized for high-performance cloud applications.
Real-time Example: I have worked on both legacy MVC systems and modern .NET Core Web APIs, allowing me to handle migrations and new feature development effectively[cite: 1].
Database 12. What are ACID properties in the context of .NET?
ACID (Atomicity, Consistency, Isolation, Durability) ensures that database transactions are processed reliably. In .NET, this is often managed using TransactionScope.
Real-time Example: I ensure ACID compliance when handling complex e-commerce checkouts where multiple tables (Orders, Inventory, Payments) must update together or not at all[cite: 1].
