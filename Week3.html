Angular 1. What are "Interceptors" in Angular and what is their use case?
Interceptors sit between your application and the backend. They can transform outgoing requests (like adding an auth token) or incoming responses (handling errors globally).
Real-time Example: I implemented an AuthInterceptor to automatically attach JWT tokens to every HTTP request in my E-commerce project[cite: 1].
Software Design 2. Can you explain the SOLID principles with an example?
SOLID stands for Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, and Dependency Inversion. They ensure code is maintainable and scalable.
Real-time Example: I apply Dependency Inversion by injecting interfaces rather than concrete classes, making my .NET Core controllers easier to unit test[cite: 1].
Angular 3. What is the difference between Reactive Forms and Template-driven Forms?
Reactive Forms are more robust and scalable, with validation handled in the TypeScript code. Template-driven Forms are easier to use for simple forms with validation in the HTML.
Real-time Example: At Cognine, I used Reactive Forms to build complex dynamic forms with custom validations for the RAB application[cite: 1].
Architecture 4. What is a "Bounded Context" in Domain-Driven Design (DDD)?
A Bounded Context defines the boundaries within which a particular domain model is defined and applicable. It helps prevent a single model from becoming overly complex.
Real-time Example: In my research on backend architecture, I've used Bounded Contexts to separate "Orders" from "Inventory" logic to keep microservices decoupled[cite: 1].
UI/UX 5. How do you handle "State Management" in Angular?
For small apps, I use Services and RxJS Subjects. For larger, complex applications, libraries like NgRx (Redux pattern) help manage global state.
Real-time Example: In the E-commerce app, I used a shared service with BehaviorSubject to manage the shopping cart state across different components[cite: 1].
Cloud Patterns 6. What is the "Circuit Breaker" pattern in Microservices?
It prevents a service from repeatedly trying to call a failing service, which can lead to cascading failures. After a threshold of failures, the "circuit" opens and returns an error immediately.
Real-time Example: I've explored using libraries like Polly in .NET to implement circuit breakers for service-to-service communication[cite: 1].
Angular 7. What are "Observables" vs "Promises"?
Promises handle a single event and are not cancellable. Observables (RxJS) handle a stream of events over time and can be cancelled.
Real-time Example: I use Observables for search input fields to debounce the user's typing, reducing unnecessary API calls to the server[cite: 1].
Performance 8. What is "Change Detection" in Angular?
It is the mechanism Angular uses to sync the UI with the data. Using ChangeDetectionStrategy.OnPush can improve performance by only checking components when their inputs change.
Real-time Example: For the data-heavy dashboards in the RAB application, I optimized performance by using the OnPush strategy where applicable[cite: 1].
C# / .NET 9. Explain the "Repository Pattern" and its benefits.
It abstracts the data access layer, allowing the business logic to stay independent of the database technology (e.g., SQL Server vs. NoSQL).
Real-time Example: I implemented the Repository Pattern in the E-commerce project to make the code cleaner and allow for easier unit testing of services[cite: 1].
DevOps 10. What is "Infrastructure as Code" (IaC)?
IaC allows you to manage and provision your cloud infrastructure (Azure VMs, Databases) using configuration files (like ARM templates or Bicep) rather than manual portal clicks.
Real-time Example: As an Azure Developer Associate, I am familiar with using scripts to ensure environment consistency between Dev, QA, and Production[cite: 1].
Frontend 11. What is the difference between "ViewChild" and "ContentChild" in Angular?
ViewChild accesses elements/components within the component's own template. ContentChild accesses elements projected into the component via <ng-content>.
Real-time Example: I used ViewChild to programmatically trigger focus on a specific input field after a form validation error[cite: 1].
Security 12. What is "Cross-Site Scripting (XSS)" and how does .NET/Angular prevent it?
XSS is when an attacker injects scripts into a webpage. Angular automatically sanitizes values, and .NET Core APIs can use libraries like HtmlEncoder to protect inputs.
Real-time Example: I always validate and sanitize user inputs in the ELOGR project to prevent malicious scripts from being saved into the database[cite: 1].
