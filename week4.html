Week 4: Scalability, Quality & Soft Skills

Azure SQL 1. How do you handle database scaling in Azure for high-traffic applications?
I leverage Azure SQL Auto-scaling and DTU/vCore adjustments. For high availability, I configure Failover Groups to ensure the database remains accessible during regional outages.
Real-time Example: In my E-commerce project, I implemented failover groups and auto-scaling to maintain performance during peak traffic periods.
Testing 2. What is the "Red-Green-Refactor" cycle in Unit Testing?
It is a TDD (Test Driven Development) approach: write a failing test (Red), write the minimum code to pass it (Green), and then optimize the code (Refactor) while keeping the test passing.
Real-time Example: At Cognine, I collaborated with manual and automation testers to ensure every new form in the RAB application was backed by reliable unit tests.
Performance 3. How do you identify bottlenecks in a .NET Core application?
I use tools like Visual Studio Profiler or Application Insights to monitor CPU usage, memory leaks, and slow-running external API calls.
Real-time Example: By profiling the ELOGR project, I identified that the Word document generation was the primary bottleneck and optimized it using the Open XML SDK.
Azure 4. When would you use "Redis Cache" over In-Memory caching?
In-Memory cache is tied to a single server instance. Redis is a distributed cache that allows multiple instances of an Azure App Service to share the same cached data.
Real-time Example: For scalable web applications, I use distributed caching to store session data so users stay logged in even if the Azure App Service scales out to multiple instances[cite: 1].

Optimization 6. Why is Open XML SDK better than Office Interop for server-side tasks?
Office Interop requires Word to be installed on the server, which is slow and not thread-safe. Open XML SDK is a lightweight library that parses the XML structure directly, making it highly efficient for server environments.
Real-time Example: This technical choice allowed me to reduce report generation time by 70% in the ELOGR project[cite: 1].
C# 7. What is "Thread Pool Starvation" and how can async/await prevent it?
Starvation occurs when all threads are blocked waiting for I/O. Using async/await releases the thread back to the pool while waiting, allowing the server to handle more concurrent requests.
Real-time Example: I consistently apply asynchronous programming in .NET Core Web APIs to ensure the application remains responsive under heavy loads[cite: 1].
